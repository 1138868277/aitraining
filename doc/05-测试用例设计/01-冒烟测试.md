# 冒烟测试

冒烟测试用于验证系统核心功能是否可正常运行，每次部署或环境变更后优先执行。

| 编号 | 场景 | 前置条件 | 步骤 | 预期结果 |
|------|------|---------|------|---------|
| ST-01 | 后端服务启动 | Node.js ≥ 24.0，pnpm ≥ 9.0，数据库连接正常 | 1. 进入 `code/apps/backend` 目录<br>2. 执行 `pnpm dev` | 终端输出服务启动信息，监听端口 3000 |
| ST-02 | 健康检查接口 | 后端服务已启动 | 调用 `GET /api/health` | 返回 `{ "code": 0, "message": "success", "data": { "status": "UP" } }` |
| ST-03 | 版本信息接口 | 后端服务已启动 | 调用 `GET /api/version` | 返回版本号和系统名称，version 为 "1.0.0" |
| ST-04 | 前端服务启动 | Node.js ≥ 24.0，依赖已安装 | 1. 进入 `code/apps/frontend` 目录<br>2. 执行 `pnpm dev` | 终端输出 Vite dev server 地址，监听端口 5173 |
| ST-05 | 登录页面可访问 | 前后端服务均已启动 | 浏览器访问 `http://localhost:5173/login` | 页面正常加载，显示用户名和密码输入框及"登录"按钮 |
| ST-06 | 未登录自动跳转 | 前后端服务均已启动 | 浏览器访问 `http://localhost:5173/code-generate` | 自动跳转到 `/login` 页面 |
| ST-07 | 登录成功 | 后端服务已启动，默认用户 admin/admin 存在 | 1. 访问登录页<br>2. 输入用户名 admin、密码 admin<br>3. 点击"登录" | 登录成功，跳转至 `/code-generate` |
| ST-08 | 编码生成页面可访问 | 已登录 | 浏览器访问 `http://localhost:5173/code-generate` | 页面正常加载，显示"快速检索"工具栏、手动/自动编码生成面板、编码预览/列表结果区 |
| ST-09 | 编码校验页面可访问 | 已登录 | 浏览器访问 `http://localhost:5173/code-verify` | 页面正常加载，左侧显示"编码解析"、"重复编码校验"、"编码修正"三个标签页 |
| ST-10 | 字典管理页面可访问 | 已登录 | 浏览器访问 `http://localhost:5173/code-validate` | 页面正常加载，显示"编码字典查询"、"字典新增"、"场站维护"三个标签页 |
| ST-11 | 统计分析页面可访问 | 已登录 | 浏览器访问 `http://localhost:5173/statistics` | 页面正常加载，显示"编码生成统计"和"编码测点分析"两个标签页 |
| ST-12 | 时序规则页面可访问 | 已登录 | 浏览器访问 `http://localhost:5173/time-series-rules` | 页面正常加载，显示流水线步骤：导入场站数据 → 导入测点数据 → 生成规则 → 导出 |
| ST-13 | 系统设置页面可访问 | 已登录为 admin 用户 | 浏览器访问 `http://localhost:5173/system-settings` | 页面正常加载，显示"数据源管理"、"用户管理"、"场站管理"、"测点导入"四个标签页 |
| ST-14 | 根路由自动跳转 | 已登录 | 浏览器访问 `http://localhost:5173/` | 自动跳转到 `/code-generate` |
| ST-15 | 字典数据加载 | 已登录，数据库有字典数据 | 1. 访问编码生成页面<br>2. 查看"类型"下拉选择框 | 下拉选项加载正常，包含 F 风电、G 光伏、S 水电、Y 不区分类型等选项 |
| ST-16 | API 代理连通 | 已登录，前后端服务均已启动 | 前端页面发起任意 API 请求 | 浏览器 Network 面板显示请求到 `/api/...`，状态码 200，返回 JSON 数据 |
| ST-17 | 退出登录 | 已登录 | 1. 点击退出登录按钮 | 跳转至 `/login`，再次访问受保护页面时跳回登录页 |
