# 冒烟测试

冒烟测试用于验证系统核心功能是否可正常运行，每次部署或环境变更后优先执行。

| 编号 | 场景 | 前置条件 | 步骤 | 预期结果 |
|------|------|---------|------|---------|
| ST-01 | 后端服务启动 | Node.js ≥ 24.0，pnpm ≥ 9.0，数据库连接正常 | 1. 进入 `code/apps/backend` 目录<br>2. 执行 `pnpm dev` | 终端输出服务启动信息，监听端口 3000 |
| ST-02 | 健康检查接口 | 后端服务已启动 | 调用 `GET /api/health` | 返回 `{ "code": 0, "message": "success", "data": { "status": "UP" } }` |
| ST-03 | 版本信息接口 | 后端服务已启动 | 调用 `GET /api/version` | 返回版本号和系统名称，version 为 "1.0.0" |
| ST-04 | 前端服务启动 | Node.js ≥ 24.0，依赖已安装 | 1. 进入 `code/apps/frontend` 目录<br>2. 执行 `pnpm dev` | 终端输出 Vite dev server 地址，监听端口 5173 |
| ST-05 | 编码生成页面可访问 | 前后端服务均已启动 | 浏览器访问 `http://localhost:5173/code-generate` | 页面正常加载，显示"快速检索"、"编码生成"、"编码结果"三个卡片区域 |
| ST-06 | 编码校验页面可访问 | 前后端服务均已启动 | 浏览器访问 `http://localhost:5173/code-validate` | 页面正常加载，显示"字典查询"、"编码解析"、"编码校验"等标签页 |
| ST-07 | 统计分析页面可访问 | 前后端服务均已启动 | 浏览器访问 `http://localhost:5173/statistics` | 页面正常加载，显示"编码生成统计"和"全量测点统计"两个标签页 |
| ST-08 | 系统设置页面可访问 | 前后端服务均已启动 | 浏览器访问 `http://localhost:5173/system-settings` | 页面正常加载，显示数据源管理表单（锁定状态） |
| ST-09 | 根路由自动跳转 | 前后端服务均已启动 | 浏览器访问 `http://localhost:5173/` | 自动跳转到 `/code-generate` |
| ST-10 | 字典数据加载 | 后端服务已启动，数据库有字典数据 | 1. 访问编码生成页面<br>2. 查看"类型"下拉选择框 | 下拉选项加载正常，包含 F 风电、G 光伏、S 水电、Y 不区分类型等选项 |
| ST-11 | API 代理连通 | 前后端服务均已启动 | 前端页面发起任意 API 请求 | 浏览器 Network 面板显示请求到 `/api/...`，状态码 200，返回 JSON 数据 |
