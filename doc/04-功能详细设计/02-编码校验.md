# 编码校验

## 功能目标

- 提供 31 位测点编码的智能解析，反向展示各段位对应的字典含义
- 支持批量粘贴和上传 Excel 两种导入方式，对编码进行字典段位识别
- 对未识别字典项提供手动新增入口，补充缺失字典数据
- 支持重复编码稽核：批量检测编码是否已在全量测点中存在
- 支持批量编码修正：通过 Excel 导入修正数据，对编码指定段位进行批量修改

## 页面说明

### 页面路径

- `/code-verify` — 编码校验主页面

### 页面布局

页面采用单卡片多标签页布局，共 4 个标签页。

#### Tab1 — 编码解析

**输入区**
- 31 位编码输入框（textarea，maxlength=31），实时显示已输入位数
- 输入自动过滤非字母数字字符
- "解析"按钮（编码满 31 位后启用）

**解析结果展示**
- 编码分段展示：以段位块的形式展示各段编码、对应的标签和名称
- 未识别段位块背景标红
- 编码可视化：以不同颜色展示各段在完整编码中的分布
- 识别状态标签："全部识别"（绿色）或"部分未识别"（红色）
- 未识别段显示错误提示

#### Tab2 — 编码校验

**输入区域**
- 两种输入模式切换：
  - 批量粘贴：textarea 输入编码列表（每行一条：编码 名称），自动解析编码和名称
  - 上传 Excel：拖拽/点击上传 .xlsx/.xls 文件，自动读取"测点编码"和"测点描述"列
- 显示已解析编码数量
- 单次上限 10000 条
- 操作按钮："开始校验"、"清空"

**校验结果**
- 展开式表格（el-table），按类型/二级类码/数据类码/数据码分组展示
- 每行展示：类型、二级类码、数据类码、数据码的编码和名称
- 展开行展示该分组下的具体测点编码列表
- 未识别的段位显示"未识别"红色标记
- 操作列：对未识别但可新增的项显示"新增"按钮
- 已处理项标记"已处理"；全部已识别的行标记"已识别"
- 仅数据码未识别的行可进行"新增"操作

#### Tab3 — 重复编码稽核

**输入区**
- textarea 粘贴编码列表（每行一条）
- 显示已解析编码数量
- 单次上限 1000 条
- 操作按钮："开始稽核"、"清空"

**稽核结果**
- 表格展示逐条编码及稽核结果：
  - 绿色标记"✓ 不重复"，说明"该编码在全量测点中不存在，可以使用"
  - 红色标记"✗ 重复"，说明"该编码已在全量测点中存在，请勿重复添加"
- 顶部汇总标签：全部不重复（绿色）或 N 条重复（红色）

#### Tab4 — 编码修正

**导入区**
- 拖拽/点击上传 .xlsx/.xls 文件
- 文件需包含三列："测点编码"、"测点描述"、"修改内容"
- 修改内容格式示例：`二级类码修改为003，数据类码修改为11，数据码修改为005`
- 单次上限 1000 条
- 显示已解析的待修正编码数量
- 操作按钮："开始修正"、"清空"

**修正进度**
- 超过 50 条自动分批处理（每批 50 条），显示 el-progress 进度条
- 显示"正在修正第 N/M 条"进度文本

**修正结果**
- 展开式表格展示每条修正结果：
  - 测点编码（旧）：31 位编码，修改的段位以红色删除线标记
  - 测点编码（新）：31 位编码，修改的段位以绿色加粗标记
  - 测点描述
  - 修改内容原文
  - 重复校验结果："✓ 不重复"或"✗ 重复"
  - 修正时间
- 顶部汇总标签：如有重复显示 N 条重复警告
- "导出 Excel"按钮：导出含新旧编码、修改内容、重复校验结果的完整报表

## 接口说明

| 接口名称 | 方法 | 路径 | 说明 |
|---------|------|------|------|
| 解析编码 | POST | /api/dict/parse-code | 反向解析 31 位编码含义 |
| 解析编码段 | POST | /api/validate/resolve-codes | 批量解析编码段位字典信息 |
| 批量检测重复 | POST | /api/statistics/measurement/check-codes | 批量检测编码是否在测点中存在 |
| 批量修正编码 | POST | /api/validate/correct-codes | 按修改内容批量修正编码段位 |
| 手动新增字典 | POST | /api/dict/manual-code | 手动新增单个字典项 |

## 业务规则

### 编码解析规则

- 输入必须为 31 位字母/数字组合
- 按预定义 11 个段位分割编码，逐个段位查询字典表
- 已识别段显示编码和名称，未识别段标记"未识别"
- 编码可视化用不同颜色区分各段位

### 校验规则

- 输入编码列表按类型/二级类码/数据类码/数据码分组展示
- 各段位逐一查询字典表，未命中标记"未识别"
- 自动去重：相同编码合并展示
- 识别完整（类型+二级类码+数据类码+数据码均已识别）的行标记"已识别"
- 仅数据码未识别的行可进行"新增"操作

### 重复编码稽核规则

- 输入编码列表逐条查询 `cec_new_energy_measurement_points` 表
- 编码在表中存在（`if_delete = '0'`）标记为重复
- 单次稽核上限 1000 条

### 编码修正规则

- 修改内容支持指定多个段位的同时修改，各修改指令以逗号分隔
- 支持的段位：场站编码、类型编码、项目期号、前缀号、一级类码、二级类码、二级类扩展码、三级类码、三级类扩展码、数据类码、数据码
- 修改内容格式：`{段位名称}修改为{新值}`
- 超过 50 条自动分批处理，每批 50 条
- 修正后的编码自动与 `cec_new_energy_measurement_points` 比对，标记是否重复
- 修正结果存入 `cec_new_energy_code_correction` 表
- 单次修正上限 100000 条

| 关联表 | 用途 |

| cec_new_energy_second_class_type_dict | 二级类码数据、校验匹配 |
| cec_new_energy_third_class_dict | 三级类码数据、校验匹配 |
| cec_new_energy_type_dict | 类型域数据 |

## 异常处理

| 异常场景 | 前端提示 |
|---------|---------|
| 编码解析输入不足 31 位 | "编码必须为31位"，解析按钮禁用 |
| 校验编码列表为空 | "请先输入待校验的编码" |
| 上传文件格式错误 | "文件格式不合法，请上传.xlsx或.xls文件" |
| 校验数量超限 | "单次解析数量超出限制（上限10000条）" |
| 稽核数量超限 | "单次稽核数量超出限制（上限1000条）" |
| 修正数量超限 | "单次修正数量超出限制（上限100000条）" |
| 修正内容格式无法解析 | "修改内容格式无法识别，请参考模板格式重新填写" |
| 稽核失败 | "稽核失败" + 错误详情 |
| 修正失败 | "编码修正失败" + 错误详情 |

## 验收标准

1. 编码解析输入 31 位编码后正确展示各段位含义
2. 编码可视化颜色分段展示清晰，未识别段标红
3. 批量粘贴自动解析编码和名称，数量显示正确
4. Excel 上传正确读取测点编码和测点描述列
5. 校验结果按分组展开展示，段位未识别正确标记
6. 未识别项"新增"按钮弹出对话框，填写保存后字典更新
7. 重复编码稽核输入编码列表后正确标记重复/不重复
8. 编码修正上传 Excel 后正确解析修改内容
9. 修正结果正确展示新旧编码对比和段位变更高亮
10. 修正结果中的重复校验正确标记
11. 修正结果可导出 Excel
12. 批量修正显示进度条
