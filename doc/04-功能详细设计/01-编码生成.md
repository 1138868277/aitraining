# 用户登录

## 功能目标

- 实现基于用户名和密码的登录认证功能
- 支持JWT令牌生成和验证
- 提供安全的会话管理机制
- 支持密码加密存储和验证

## 页面或接口说明

- 登录页面：/login - 前端Vue组件，包含用户名和密码输入框
- 登录接口：POST /api/auth/login - 后端登录API
- 令牌刷新接口：POST /api/auth/refresh - 刷新JWT令牌
- 退出接口：POST /api/auth/logout - 用户登出

## 输入输出

- 输入：
  - username: string (用户名，必填)
  - password: string (密码，必填)
- 输出：
  - token: string (JWT令牌)
  - refreshToken: string (刷新令牌)
  - expiresIn: number (令牌过期时间，秒)
  - user: object (用户基本信息)

## 业务规则

- 用户名和密码必填，长度限制：用户名4-20字符，密码6-20字符
- 密码使用BCrypt加密存储，登录时进行匹配验证
- 登录成功生成JWT令牌，设置30分钟过期时间
- 刷新令牌有效期为7天，用于续期主令牌
- 同一账号同一时间只允许一个活跃会话
- 登录失败超过5次触发账户锁定（15分钟）

## 异常处理

- 用户名或密码错误：返回400状态码，错误码LOGIN_FAILED
- 账户被锁定：返回403状态码，错误码ACCOUNT_LOCKED
- 令牌过期：返回401状态码，提示重新登录
- 无效令牌：返回401状态码，错误码INVALID_TOKEN
- 系统异常：返回500状态码，记录详细错误日志

## 验收标准

- 成功登录返回有效JWT令牌
- 令牌包含必要claims（用户ID、角色、过期时间）
- 密码错误3次后显示验证码
- 登录成功后前端存储令牌并跳转至首页
- 令牌过期前可成功刷新
- 多次登录失败触发账户锁定机制
- 登出后令牌立即失效
