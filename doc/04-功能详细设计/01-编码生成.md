# 编码生成

## 功能目标

- 提供基于字典的筛选条件面板，支持快速检索字典项和手动补充缺失字典
- 根据用户选定的条件组合，自动生成合规的 31 位测点编码及编码名称
- 支持单条生成和批量生成（扩展码范围组合）
- 支持草稿临时保存、最近筛选条件复用、编码历史记录管理
- 支持编码的复制、Excel 导出和数据库入库持久化

## 页面说明

### 页面路径

- `/code-generate` — 编码生成主页面

### 页面布局

页面采用上下分块布局，三大卡片依次排列：

**卡片一：快速检索**
- 顶部：标题"快速检索" + "新增"按钮（弹出手动新增字典对话框）
- 搜索行：输入框（支持数据码模糊搜索或编码后5位精确搜索）+ 最近搜索标签
- 搜索结果表格：列包含类型域、二级类码、数据类码、数据码、来源（集团统一/手动添加）、操作（"筛选"按钮，点击自动填充生成条件）
- 支持类型和二级类码筛选、分页控制

**卡片二：编码生成（筛选条件面板）**
- 顶部：标题"编码生成" + "最近记录"按钮（弹出最近筛选条件列表，点击自动填充）
- 筛选条件表单，每行 4 个条件项，字段类型分为下拉选择、文本输入、扩展数输入三种：
  - 场站（stationCode）：下拉选择，单选，支持锁定
  - 发电类型（typeCode）：下拉选择，单选，支持锁定
  - 二级类码（secondClassCode）：下拉选择，支持多选，支持锁定
  - 三级类码（thirdClassCode）：下拉选择，单选
  - 数据类码（dataTypeCode）：下拉选择，单选
  - 数据码（dataCode）：下拉选择
  - 二级扩展码（secondExtCode）：扩展数字输入（起始值 + 延伸数量），默认从 1 开始
  - 三级扩展码（thirdExtCode）：扩展数字输入（起始值 + 延伸数量），默认从 0 开始
- 条件锁定功能：点击锁定按钮可固定某个条件值，切换其他条件时该值不变
- 快捷选项：部分字段提供快捷标签（如"不限"）
- 操作按钮：生成编码、清空条件
- 多选 + 延伸数量组合时显示预计生成编码数

**卡片三：编码结果（含两个标签页）**
- Tab1 — 编码预览：展示最近一次生成的编码列表，支持复制全部、保存入库、单条复制、行内编辑编码名称
- Tab2 — 编码列表：展示已入库保存的编码历史记录，支持类型/场站/二级类码等筛选条件、分页、导出（Excel）、单条/批量删除

## 接口说明

| 接口名称 | 方法 | 路径 | 说明 |
|---------|------|------|------|
| 单条生成编码 | POST | /api/codes/generate | 根据筛选条件生成单条编码 |
| 批量生成编码 | POST | /api/codes/batch-generate | 多条件+扩展码批量生成 |
| 保存草稿 | POST | /api/codes/draft | 将编码保存至临时草稿区 |
| 获取草稿列表 | GET | /api/codes/draft | 获取临时草稿区列表 |
| 删除单条草稿 | DELETE | /api/codes/draft/:id | 删除指定草稿 |
| 清空草稿 | DELETE | /api/codes/draft/batch | 清空全部草稿 |
| 编码入库 | POST | /api/codes | 将编码保存至数据库 |
| 查询历史记录 | GET | /api/codes | 分页查询已入库的编码记录 |
| 删除单条记录 | DELETE | /api/codes/:id | 删除指定历史记录 |
| 批量删除记录 | DELETE | /api/codes/batch | 批量删除历史记录 |
| 导出历史记录 | GET | /api/codes/export | 导出编码历史记录为 Excel |
| 保存最近条件 | POST | /api/codes/recent-conditions | 保存最近使用的筛选条件 |
| 获取最近条件 | GET | /api/codes/recent-conditions | 获取最近筛选条件列表 |

## 输入输出

### 生成编码

**输入：**

```json
{
  "stationCode": "string",
  "typeCode": "string",
  "secondClassCode": "string | string[]",
  "thirdClassCode": "string",
  "dataTypeCode": "string",
  "dataCode": "string",
  "secondExtCodeStart": "number",
  "secondExtCodeCount": "number",
  "thirdExtCodeStart": "number",
  "thirdExtCodeCount": "number"
}
```

**输出：**

```json
{
  "codes": [
    {
      "code": "ABCDEFGHIJKLMNOPQRSTU12345678901",
      "name": "编码名称",
      "generateTime": "2025-01-01 12:00:00"
    }
  ]
}
```

### 保存草稿

**输入：**

```json
{
  "codes": [
    { "code": "string", "name": "string" }
  ]
}
```

**输出：**

```json
{
  "savedCount": 10,
  "totalCount": 15
}
```

## 业务规则

### 编码规则

- 测点编码由多段编码维度拼接而成，共 31 位
- 编码段位构成：场站编码(4位) + 类型(2位) + 项目线(3位) + 前缀(1位) + 一级类码(2位) + 二级类码(3位) + 二级扩展码(4位) + 三级类码(3位) + 三级扩展码(4位) + 数据分类码(2位) + 数据码(3位)
- 编码名称由各段字典项名称拼接生成

### 条件筛选规则

- 必选条件：至少场站、发电类型、二级类码必须选择
- 条件联动：选择上级后下级自动加载对应选项（如选类型后二级类码列表/数据类码联动）
- 功能锁定：条件可单独锁定，切换其他条件时锁定值不变，方便固定某些维度调整其他维度生成

### 批量生成规则

- 二级类码支持多选，多个值自动进行笛卡尔积组合生成多条编码
- 二级扩展码和三级扩展码支持设置起始值和延伸数量，自动生成连续编号范围内的编码
- 批量生成时显示预计生成编码数量
- 单次生成上限 100 条

### 草稿（临时区）规则

- 草稿数据存储于后端内存（以 x-session-id 隔离），服务重启后丢失
- 草稿上限 5000 条
- 支持多轮生成后统一处理（入库或导出）

### 最近条件规则

- 每次生成编码后，当前的筛选条件组合自动保存为最近条件
- 最近条件列表在"最近记录"弹出面板中展示，显示各字段值和摘要
- 点击最近条件项自动填充筛选条件并触发生成

### 导出规则

- 导出文件格式：.xlsx
- 单次导出上限：10000 行

## 异常处理

| 异常场景 | 前端提示 |
|---------|---------|
| 字典数据加载失败 | "筛选条件加载失败，请刷新重试" |
| 必选条件未选全 | 生成按钮置灰不可点击 |
| 单次生成超限 | "单次生成数量超出限制（上限100条）" |
| 草稿区已满 | "临时区已满（上限5000条），请先导出或清理" |

## 验收标准

1. 快速检索支持关键字搜索和类型/二级类码筛选，结果正确
2. 手动新增字典对话框填写保存后字典数据更新
3. 最近记录点击后正确填充筛选条件
4. 筛选条件联动正确，锁定功能生效
5. 生成编码为 31 位，段位构成正确
6. 批量生成（多选+扩展码）编码数量计算正确
7. 编码预览列表正确展示，支持复制、保存
8. 编码列表（历史记录）筛选、分页、导出、删除正常
