# 异常处理与边界场景

## 通用异常

| 异常类型 | HTTP状态码 | 错误码 | 提示信息 |
|---------|-----------|-------|---------|
| 参数缺失 | 400 | MISSING_PARAMETER | 请填写必填参数：{字段名} |
| 参数格式错误 | 400 | PARAM_FORMAT_ERROR | 参数{字段名}格式不正确 |
| 资源不存在 | 404 | RESOURCE_NOT_FOUND | 请求的资源不存在 |
| 重复提交 | 409 | DUPLICATE_SUBMISSION | 请勿重复提交 |
| 鉴权失败 | 401 | AUTHENTICATION_FAILED | 未登录或登录已过期 |
| 权限不足 | 403 | PERMISSION_DENIED | 无权执行该操作 |
| 请求超时 | 504 | REQUEST_TIMEOUT | 请求超时，请重试 |
| 系统异常 | 500 | SYSTEM_ERROR | 系统繁忙，请稍后重试 |
| 外部服务异常 | 503 | EXTERNAL_SERVICE_ERROR | 外部服务暂不可用，请稍后重试 |

## 边界场景

### 数据边界

| 场景 | 处理方式 |
|-----|---------|
| 空数据 | 返回200状态码，data为空数组或null，前端展示空状态提示 |
| 超大数据量 | 超出模块限制时提示分批处理，前端控制上传/输入上限 |
| 特殊字符 | 编码中不允许出现特殊字符，输入时前端校验过滤 |
| 前后端数据不一致 | 返回409状态码，提示"数据已被修改，请刷新重试" |
| 并发提交 | 返回409状态码，提示"请勿重复提交" |

### 操作边界

| 场景 | 处理方式 |
|-----|---------|
| 用户未选条件点击生成 | 生成按钮置灰，不可点击 |
| 用户未输入编码点击校验 | 校验按钮置灰，不可点击 |
| 用户未上传文件查看统计 | 图表区域显示空状态引导提示 |
| 页面刷新 | 临时区数据丢失，提示用户先保存至数据库 |
| 浏览器兼容性 | 复制功能使用Clipboard API，不兼容时提示手动复制 |
| 网络断开后重连 | 保留当前页面状态，自动恢复数据加载 |

### 输入校验边界

| 字段 | 校验规则 |
|-----|---------|
| 编码格式 | 必须符合31位编码格式，只能包含字母和数字 |
| 编码列表 | 每行一条，去除首尾空格和空行 |
| 上传文件 | 仅支持.xlsx/.xls，大小不超过10MB |
| 文件内容 | 必须包含编码、编码名称、管理域等必要字段 |

## 处理原则

- **统一异常处理**：后端使用 @RestControllerAdvice 全局处理异常，返回标准 ErrorResponse 格式
- **用户友好提示**：业务异常提供清晰的中文提示，系统异常提供通用提示，不暴露内部细节
- **详细错误日志**：记录异常堆栈、请求参数、用户信息等上下文，便于排查
- **幂等性设计**：关键操作确保幂等（如数据导入、一键纠错），防止重复提交影响
- **超时控制**：设置合理的超时时间（批量校验30秒、数据导入60秒），避免长时间等待
- **前端防重复**：提交后按钮置灰并显示加载状态，防止重复点击

## 通用响应格式

### 成功响应

```json
{
  "code": 0,
  "message": "success",
  "data": {},
  "timestamp": 1640995200000
}
```

### 错误响应

```json
{
  "code": 10001,
  "message": "参数校验失败",
  "data": {
    "field": "stationCode",
    "reason": "不能为空"
  },
  "timestamp": 1640995200000
}
```

### 错误码分段

| 错误码范围 | 模块 |
|-----------|------|
| 10xxx | 编码生成模块 |
| 20xxx | 编码校验模块 |
| 30xxx | 统计分析模块 |
| 90xxx | 通用错误码（参数校验、权限等） |
