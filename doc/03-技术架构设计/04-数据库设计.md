# 数据库设计

## 文档定位

本文件只描述当前项目自己的数据库设计结果，不重复数据库通用规范。

## 设计范围

- 业务实体与表关系
- 核心表清单（12 张表）
- 关键字段设计
- 索引方案
- 与外部服务的数据边界

## 数据实体

核心业务实体包含 5 类：

1. **字典实体**：类型域、前缀、项目线、编码字典、数据类型、二级类码、三级类码
2. **编码生成记录**：保存生成的测点编码及上下文
3. **场站字典**：新能源场站基础信息
4. **校验数据**：导入的编码稽核数据
5. **全量测点数据**：存量测点导入数据

## 核心表详情

### 1. 类型域字典（cec_new_energy_type_dict）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| type_id | BIGSERIAL | 主键 |
| type_code | VARCHAR(2) | 类型域编码（F:风电, G:光伏, Y:水电） |
| type_name | VARCHAR(100) | 类型域名称 |

### 2. 编码字典（cec_new_energy_code_dict）— 核心字典表

| 字段名 | 类型 | 说明 |
|--------|------|------|
| code_dict_id | BIGSERIAL | 主键 |
| type_domain_code | CHAR(1) | 类型域编码（F/G/Y），CHECK约束 |
| first_class_code | VARCHAR(2) | 一级类码 |
| first_class_name | VARCHAR(100) | 一级类码名称 |
| second_class_code | VARCHAR(3) | 二级类码 |
| second_class_name | VARCHAR(100) | 二级类码名称 |
| data_category_code | VARCHAR(2) | 数据分类码 |
| data_category_name | VARCHAR(100) | 数据分类名称 |
| data_code | VARCHAR(3) | 数据码 |
| data_name | VARCHAR(100) | 数据名称 |
| is_manual | CHAR(1) | 是否手动新增（'0'否, '1'是） |
| type_code | VARCHAR(10) | 冗余的类型域编码 |

该表是编码规则的核心，定义了 5 级分类体系：类型域 → 一级类码 → 二级类码 → 数据分类码 → 数据码。

### 3. 编码生成记录（cec_new_energy_createcode）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | BIGSERIAL | 主键 |
| code | VARCHAR(31) | 31 位测点编码 |
| name | VARCHAR(500) | 编码名称 |
| batch_no | VARCHAR(20) | 生成批次号 |
| type_code | VARCHAR(100) | 生成时的类型域编码 |
| second_class_code | VARCHAR(100) | 生成时的二级类码 |
| data_type_code | VARCHAR(100) | 生成时的数据类型编码 |
| station_code | VARCHAR(100) | 生成时的场站编码 |
| third_class_code | VARCHAR(100) | 生成时的三级类码 |

### 4. 全量测点数据（cec_new_energy_measurement_points）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | BIGSERIAL | 主键 |
| code | VARCHAR(31) | 31 位测点编码 |
| station_code | VARCHAR(4) | 场站编码（1-4位） |
| type_code | VARCHAR(2) | 类型域编码（第5位） |
| project_line_code | VARCHAR(3) | 项目线编码（第6-8位） |
| prefix_no | VARCHAR(1) | 前缀编号（第9位） |
| first_class_code | VARCHAR(2) | 一级类码（第10-11位） |
| second_class_code | VARCHAR(3) | 二级类码（第12-14位） |
| second_ext_code | VARCHAR(4) | 二级扩展码（第15-18位） |
| third_class_code | VARCHAR(3) | 三级类码（第19-21位） |
| third_ext_code | VARCHAR(4) | 三级扩展码（第22-25位） |
| data_category_code | VARCHAR(2) | 数据分类码（第26-27位） |
| data_code | VARCHAR(3) | 数据码（第29-31位） |
| import_batch_id | VARCHAR(50) | 导入批次 ID |

该表存储从 Excel 导入的存量测点数据，将 31 位编码解析为各段位分别存储，便于分段聚合统计。

### 5. 最近筛选条件（cec_new_energy_recent_condition）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | BIGSERIAL | 主键 |
| condition_data | JSONB | 筛选条件数据（JSON 格式） |
| condition_summary | VARCHAR(500) | 条件摘要描述 |

## 索引设计

### 核心索引

| 表名 | 索引名 | 列 | 说明 |
|------|--------|---|------|
| cec_new_energy_code_dict | idx_code_dict_tree_query | type_domain_code, first_class_code, second_class_code, data_category_code | 字典树查询覆盖索引 |
| cec_new_energy_createcode | idx_createcode_tm | create_tm DESC | 按时间倒序查询 |
| cec_new_energy_second_class_type_dict | idx_second_class_type | type_code, second_class_code | 按类型域查二级类码 |
| cec_new_energy_third_class_dict | idx_third_class_type_second | type_code, second_class_code | 按类型域+二级类码查三级类码 |
| cec_new_energy_measurement_points | idx_mp_type | type_code | 按类型域统计 |
| cec_new_energy_measurement_points | idx_mp_station | station_code | 按场站统计 |
| cec_new_energy_measurement_points | idx_mp_second_class | second_class_code | 按二级类码统计 |
| cec_new_energy_measurement_points | idx_mp_import_batch | import_batch_id | 按导入批次查询 |
| cec_new_energy_measurement_points | idx_mp_ifdel_crtm | if_delete, create_tm DESC | 复合过滤排序 |
| cec_new_energy_recent_condition | idx_recent_condition_tm | create_tm DESC | 按时间倒序 |

## 关键设计说明

- **主键策略**：所有表使用 BIGSERIAL（自增序列），由数据库自动生成
- **逻辑删除**：所有业务表使用 `if_delete` 字段标记删除状态，查询时默认过滤 `if_delete = '0'`
- **无外键约束**：表间无外键约束，数据完整性由应用层保证，便于批量数据操作
- **JSONB 存储**：最近筛选条件使用 JSONB 类型存储，充分利用 PostgreSQL 的 JSON 查询能力
- **编码解析存储**：全量测点数据表将 31 位编码按位段拆分存储，支撑按各维度独立聚合统计
