# 系统模块设计

## 模块划分

- 模块1：编码生成模块
- 模块2：编码校验与纠错模块
- 模块3：分析统计模块

## 模块职责

- 模块1职责：
    提供筛选条件
    根据用户选择的条件自动生成测点编码及编码名称
    支持临时保存、批量汇总、复制、导出Excel
    支持编码记录的持久化保存与历史追溯

- 模块2职责：
    支持手动输入、批量粘贴、上传Excel三种编码输入方式
    根据编码字典规则自动校验编码合规性
    对异常编码给出纠错建议（编辑距离匹配）
    展示校验结果表格，支持结果导出

- 模块3职责：
    统计系统总编码数量、合规/不合规数量及合规率
    按管理域等维度进行细分统计
    使用柱状图、饼图可视化展示统计数据
    数据来源于编码生成记录和校验记录


## 模块交互

- 模块间调用关系：
    模块1和模块2共用基础字典数据

- 关键依赖：
    模块1、2均依赖数据库中的字典表
    模块2依赖编码标准字典表进行合规性判断
