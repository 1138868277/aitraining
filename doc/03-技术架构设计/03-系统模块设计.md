# 系统模块设计

## 模块划分

系统包含 10 个功能模块，遵循 Controller → Service → Domain 三层结构：

| 模块 | 后端目录 | Controller | Service | Domain |
|------|---------|-----------|---------|--------|
| 认证 | `auth/` | `auth-controller.ts` | `auth-service.ts` | `auth-domain.ts` |
| 编码生成 | `code-generation/` | `code-controller.ts` | `code-service.ts` | `code-domain.ts` |
| 自动编码 | `auto-code/` | `auto-code-controller.ts` | `auto-code-service.ts` | `auto-code-domain.ts` |
| 编码校验 | `code-validation/` | `validate-controller.ts` | `validate-service.ts` | `validate-domain.ts` |
| 字典管理 | `dict/` | `dict-controller.ts` | `dict-service.ts` | `dict-domain.ts` |
| 字典审批 | `approval/` | `approval-controller.ts` | `approval-service.ts` | `approval-domain.ts` |
| 场站管理 | `station/` | `station-controller.ts` | `station-service.ts` | `station-domain.ts` |
| 统计分析 | `statistics/` | `statistics-controller.ts` | `statistics-service.ts` | `statistics-domain.ts` |
| 时序规则 | `tsr/` | `tsr-controller.ts` | `tsr-service.ts` | `tsr-domain.ts` |
| 数据源管理 | `datasource/` | `datasource-controller.ts` | `datasource-service.ts` | - |

## 模块职责

### 0. 认证模块（auth）

**功能**：用户登录、Token 签发与验证、用户 CRUD

**核心能力**：
- 用户名+密码登录验证，JWT Token 签发（HS256 算法，24 小时有效期）
- Token 验证与当前用户信息查询
- 用户列表查询、新增用户、编辑用户、删除用户（仅集团管理员）
- 默认预置三个用户：admin（集团管理员）/ yunnan（云南区域）/ fujian（福建区域）
- 用户数据统一存储在 admin 租户的 `cec_sys_user` 表
- 用户登录时自动更新最后登录时间

### 1. 编码生成模块（code-generation）

**功能**：编码生成引擎、草稿管理、历史记录管理、最近筛选条件、快速检索、Excel 导出

**核心能力**：
- 根据 11 类字典筛选条件实时生成合规 31 位测点编码
- 支持单条生成和批量生成（多值条件自动笛卡尔积组合）
- 生成的编码可暂存至内存草稿区（session 级别，支持多轮生成后统一处理）
- 编码可持久化入库保存，支持历史记录的分页查询、删除和 Excel 导出
- 最近使用的筛选条件自动保存，支持一键填充复用
- 嵌入快速检索面板，支持按数据码关键字模糊搜索与类型筛选（F/G/S）

### 2. 自动编码模块（auto-code）

**功能**：Excel 导入自动字典匹配与编码生成

**核心能力**：
- 上传含设备名称、场站、二级类码等字段的 Excel 文件
- 列映射向导：将 Excel 列映射到系统字段（场站名称、二级类码、三级类码等）
- 编码配置：支持项目期号、并网线路、前缀号等固定参数
- 自动字典匹配：基于映射后的字段查询字典，批量生成 31 位编码
- 数量限制：单次最多处理 20000 条编码

### 3. 编码校验模块（code-validation）

**功能**：编码合规校验、智能纠错、编码解析、重复编码稽核、编码修正

**核心能力**：
- **编码解析**：
  - 31 位编码按 11 个段位拆解，反向解析各段位对应的字典含义
  - 段位可视化展示（分块卡片 + 编码结构图）
  - 识别状态标记（全部识别 / 部分未识别）
  - 支持单条和批量两种解析模式
- **重复编码稽核**：
  - 批量粘贴编码列表，与全量测点数据比对
  - 逐条标记"重复"或"不重复"状态
  - 单次上限 10000 条
- **编码修正**：
  - Excel 导入含"测点编码、测点描述、修改内容"三列的修正数据
  - 按修改内容解析需要修改的段位，批量生成修正后编码
  - 修正前后编码按段位逐段对比展示（旧值标红删除线，新值标绿加粗）
  - 修正后编码自动与全量测点比对，标记重复风险
  - 超过 50 条自动分批处理，实时进度条反馈
  - 修正结果导出为 Excel

### 4. 字典管理模块（dict）

**功能**：字典树展示、快速检索、31 位编码解析、手动新增字典、新增字典记录

**核心能力**：
- 分层树形结构展示全部 11 类字典项，支持按类型展开浏览（数据码层懒加载）
- 关键字模糊搜索（数据码查询模式）与树形浏览模式两种切换
- 字典项概览：按风电/光伏/水电类型统计各字典维度数据量
- 31 位编码智能解析，反向展示各段位对应的字典含义
- 手动新增字典项（单条和批量），标记为手工添加
- 新增字典记录的分页查询，支持类型和二级类码筛选、Excel 导出
- 按设备类型获取二级类码列表，按数据类型获取数据码列表
- 最大数据码/分类码查询，辅助手动新增时的编码分配

### 5. 字典审批模块（approval）

**功能**：区域提交字典新增申请、集团审批、全租户数据分发

**核心能力**：
- 区域用户提交数据码/数据类码新增申请（含类型、二级类码、数据类码、数据码、名称）
- 集团管理员查看待审批列表，审核通过或驳回（需填写原因）
- 审批通过后，自动遍历所有租户 Schema 写入字典数据（已存在则跳过）
- 区域侧草稿表与 admin 侧审批表双向状态同步
- 提交记录分页查询，支持按区域和状态筛选
- 重复提交检测：同一用户对同一数据码不能重复提交

### 6. 场站管理模块（station）

**功能**：场站基础数据 CRUD、批量操作、导出

**核心能力**：
- 场站分页列表查询
- 单条新增与批量新增（含重复检测）
- 编辑与删除（单个删除和全部清空）
- Excel 导出

### 7. 统计分析模块（statistics）

**功能**：编码生成统计、全量测点统计、编码重复检测

**编码生成统计**：
- 概览统计：累计生成总数、今日/本周/本月生成数
- 类型维度：风电、光伏、水电、不区分类型的编码数量
- 二级类码维度：按设备类型筛选的柱状图统计
- 场站维度：按场站聚合的柱状图统计
- 分组详情列表：按生成批次聚合展示，展开查看单条编码
- 生成趋势：按时间维度的趋势数据

**全量测点统计**：
- Excel 大文件导入：自动解析 31 位编码并分段存储，支持进度提示和取消导入
- 概览统计：总测点数、风电/光伏/其他分类
- 二级类码维度、场站维度的柱状图统计
- 测点列表查询及筛选

**编码重复检测**（被编码校验模块的重复编码稽核调用）：
- 批量查询指定编码集合在全量测点表中的存在状态

### 8. 时序规则模块（tsr / time-series-rules）

**功能**：场站数据导入、测点数据导入、四类稽核规则生成、多格式导出

**核心能力**：
- Excel 导入场站数据和测点数据（支持 200MB 以内文件）
- 自动生成四类时序稽核规则：
  - **死值（sz）**：检测测点数据在滑动窗口内是否保持恒定值
  - **突变（tb）**：检测测点数据是否发生异常跳变
  - **越限（yx）**：检测测点数据是否超出合理上下限范围
  - **终端（zd）**：检测测点数据是否在指定时长内无更新
- 流水线式分步操作：场站导入 → 测点导入 → 规则生成 → 结果导出
- 实时进度跟踪：各步骤导入进度、生成进度百分比显示
- 多种导出方式：
  - 按规则类型分别导出 Excel 单文件
  - 全部类型打包为 ZIP 下载
  - 分批拆分导出（按 Z7 逻辑，适配大数据量场景）
  - 总体结果（单文件）与分批结果（多文件）同时提供
- 表自动创建：首次使用时通过 `CREATE TABLE IF NOT EXISTS` 自动建表
- 数据清空：一键清空所有导入的场站和测点数据

### 9. 数据源管理模块（datasource）

**功能**：PostgreSQL 数据源配置、连接测试、保存切换

**核心能力**：
- 查看当前数据源连接配置
- 测试新配置的数据库连接是否可用
- 保存配置并在运行时动态切换连接池，无需重启服务
- 配置表单锁定保护，防止误修改

### 关键依赖说明

- **字典数据共享**：编码生成、自动编码、编码解析三个模块均依赖字典管理模块维护的字典数据
- **数据源全局性**：数据源管理模块影响整个后端数据库连接，切换后将作用于所有模块
- **内存状态隔离**：编码生成的草稿数据和编码校验的校验任务均存储于后端内存，相互独立
- **前后端契约**：所有模块的请求/响应类型定义共享于 `@cec/contracts` 包，保证前后端数据格式一致
- **编码重复检测依赖**：编码校验模块的重复编码稽核和修正重复检测依赖统计分析模块的全量测点数据
- **认证全局性**：认证模块的租户上下文通过中间件注入，影响后续所有业务模块的数据库路由
- **审批分发依赖**：审批模块通过后需遍历所有租户的 Schema 写入字典数据，依赖数据源配置
- **时序规则独立性**：TSR 模块使用独立的表（`tsr_*` 前缀），与其他模块数据无依赖关系
