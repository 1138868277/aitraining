# 部署与运行设计

## 部署架构概览

本应用采用**前后端分离架构**，本地开发环境部署方案如下：

```
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│   前端开发服务器   │    │   Node.js       │    │   PostgreSQL    │
│   (Vite dev)     │◄──►│   后端服务       │◄──►│    数据库       │
│   Vue 3 + Vite  │    │   (Express)     │    │                 │
│   Port 5173     │    │   Port 3000     │    │   Port 7300     │
└─────────────────┘    └─────────────────┘    └─────────────────┘
         │                       │
         └─────── 代理 /api ─────┘
         (Vite 开发代理配置)
```

## 本地运行方式

### 环境准备

1. **Node.js 环境**：≥ 24.0
2. **包管理器**：pnpm ≥ 9.0
3. **数据库**：PostgreSQL 14+（开发环境使用远程实例 `10.1.1.113:7300`）

### 安装依赖

```bash
# 在 code 目录下执行（monorepo 根目录）
cd code

# 安装所有依赖（包含 apps/backend、apps/frontend、packages/*）
pnpm install
```

### 数据库准备

数据库为远程 PostgreSQL 实例（`10.1.1.113:7300`），Schema 为 `liuhaojun`。

建表脚本位于 `code/sql/` 目录，按编号顺序执行：
```
001-create-tables.sql      # 创建核心表
002-init-data.sql          # 初始化字典数据
003-create-test-data.sql   # 创建测试数据
004-modify-id-to-serial.sql
005-refactor-code-dict-table.sql
006-create-recent-condition-table.sql
007-add-is-manual-column.sql
008-add-dict-tree-index.sql
009-add-type-code-column.sql
010-add-save-context-columns.sql
011-widen-context-columns.sql
```

后端独立的迁移脚本：
```
apps/backend/src/db/migrations/001_create_measurement_points.sql
```

应用层动态建表（首次使用时自动创建，无需手动执行）：
- `cec_new_energy_code_mapping`（编码映射表）
- `cec_new_energy_code_correction`（编码修正记录表）

### 后端启动方式

```bash
# 启动后端开发服务（tsx watch 热更新）
cd apps/backend
pnpm dev
```

- 后端服务默认运行在：`http://localhost:3000`
- 使用 `tsx watch` 实现文件变更自动重启
- API 接口前缀：`/api`

### 前端启动方式

```bash
# 启动前端开发服务（Vite 热更新）
cd apps/frontend
pnpm dev
```

- 开发服务器默认运行在：`http://localhost:5173`
- Vite 配置中将 `/api` 请求代理到 `http://localhost:3000`
- 支持热模块替换（HMR）

### 从根目录快速启动

```bash
# 同时在根目录启动前后端
cd code

# 启动后端
pnpm dev:backend

# 启动前端（另一个终端）
pnpm dev:frontend
```

### 完整本地启动流程

1. 配置 `.env` 文件中的数据库连接信息
2. 执行数据库建表脚本和初始化数据脚本
3. 在 `code/` 目录执行 `pnpm install` 安装依赖
4. 启动后端服务 `pnpm dev:backend`
5. 启动前端服务 `pnpm dev:frontend`
6. 访问 `http://localhost:5173` 使用应用

## 构建部署

### 前端构建

```bash
cd apps/frontend
pnpm build
```

- 构建产物输出到 `apps/frontend/dist/` 目录
- 为静态文件（HTML/JS/CSS），可用 Nginx 等托管

### 后端构建

```bash
cd apps/backend
pnpm build
```

- 使用 `tsc` 编译 TypeScript 到 JavaScript
- 构建产物输出到 `apps/backend/dist/` 目录
- 使用 `node dist/main.js` 启动生产服务

### 生产部署方式

**方案一：直接部署**
- 后端：`node dist/main.js`（配置环境变量后运行）
- 前端：Nginx 托管 `dist/` 目录，配置反向代理 `/api` 到后端

**方案二：容器化部署**
- 构建 Docker 镜像，分别部署后端和前端容器
- 使用 Docker Compose 编排服务

## 环境变量说明

### 核心环境变量（`.env`）

| 变量名 | 说明 | 示例值 |
|--------|------|--------|
| `DB_HOST` | 数据库主机地址 | `10.1.1.113` |
| `DB_PORT` | 数据库端口 | `7300` |
| `DB_NAME` | 数据库名称 | `training_exercises` |
| `DB_SCHEMA` | 数据库 schema | `liuhaojun` |
| `DB_USER` | 数据库用户名 | `liuhaojun` |
| `DB_PASSWORD` | 数据库密码 | `liuhaojun` |
| `SERVER_PORT` | 后端服务端口 | `3000` |
| `FRONTEND_URL` | 前端访问地址 | `http://localhost:5173` |
| `FILE_UPLOAD_MAX_SIZE` | 文件上传最大大小(MB) | `10` |

### 数据源配置文件（`config/datasource.json`）

数据库连接配置持久化于此文件，优先级高于 `.env`。系统启动时优先读取此文件，若不存在则读取 `.env` 中的配置。运行时通过"系统设置 → 数据源管理"页面修改后自动更新到此文件。
