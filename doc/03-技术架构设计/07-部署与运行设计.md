# 部署与运行设计

## 部署架构概览

本应用采用**前后端分离架构**，部署方案设计如下：

```
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│   前端静态资源   │    │   Spring Boot   │    │   PostgreSQL    │
│   (Nginx托管)   │◄──►│   后端服务      │◄──►│    数据库       │
│   Vue 3 + Vite  │    │   (JAR包运行)   │    │   (SeaboxSQL)   │
└─────────────────┘    └─────────────────┘    └─────────────────┘
         │                       │                       │
         └───────────────────────┼───────────────────────┘
                                 │
                          ┌─────────────────┐
                          │     监控运维     │
                          │   (日志/监控/告警)│
                          └─────────────────┘
```

## 本地运行方式

### 环境准备
1. **Node.js环境**：≥ 24.0（前端开发）
2. **Java环境**：JDK 17（后端开发）
3. **数据库**：PostgreSQL 14+（或SeaboxSQL）
4. **包管理器**：pnpm ≥ 9.0（前端），Maven 3.8+（后端）

### 前端启动方式
```bash
# 进入前端项目目录
cd frontend

# 安装依赖
pnpm install

# 开发环境启动（热更新）
pnpm dev

# 构建生产版本
pnpm build
```
- 开发服务器默认运行在：`http://localhost:5173`
- 生产构建输出到：`frontend/dist` 目录

### 后端启动方式
```bash
# 进入后端项目目录
cd backend

# 编译打包
mvn clean package

# 运行Spring Boot应用
java -jar target/cec-new-energy-code-1.0.0.jar

# 或使用Maven直接运行（开发环境）
mvn spring-boot:run
```
- 后端服务默认运行在：`http://localhost:8080`
- API接口前缀：`/api`

### 数据库准备方式
1. **创建数据库**：
```sql
CREATE DATABASE cec_new_energy_code 
  ENCODING 'UTF8' 
  LC_COLLATE 'zh_CN.UTF-8' 
  LC_CTYPE 'zh_CN.UTF-8';
```

2. **执行建表脚本**：
```bash
# 执行sql目录下的初始化脚本
psql -h localhost -U postgres -d cec_new_energy_code -f code/sql/init_tables.sql
```

3. **初始化基础数据**：
```bash
# 执行字典数据初始化脚本
psql -h localhost -U postgres -d cec_new_energy_code -f code/sql/init_data.sql
```

### 完整本地启动流程
1. 启动PostgreSQL数据库服务
2. 执行数据库初始化脚本
3. 启动后端Spring Boot服务
4. 启动前端开发服务器
5. 访问 `http://localhost:5173` 使用应用

## 环境说明

### 本地开发环境
- **用途**：开发、调试、单元测试
- **部署方式**：前后端分别本地运行
- **数据库**：本地PostgreSQL实例
- **配置特点**：启用热更新、调试日志、Swagger文档

- **数据库**：独立测试数据库实例

- **用途**：正式对外服务
- **部署方式**：Kubernetes集群部署 或 传统服务器部署

- **配置特点**：性能优化、安全加固、监控告警

## 配置项

### 必填环境变量
| 变量名 | 说明 | 示例值 | 环境 |
|--------|------|--------|------|
| `DB_HOST` | 数据库主机地址 | `localhost` | 所有 |
| `DB_PORT` | 数据库端口 | `5432` | 所有 |
| `DB_NAME` | 数据库名称 | `cec_new_energy_code` | 所有 |
| `DB_USER` | 数据库用户名 | `postgres` | 所有 |
| `DB_PASSWORD` | 数据库密码 | `your_password` | 所有 |
| `SERVER_PORT` | 后端服务端口 | `8080` | 所有 |
| `FRONTEND_URL` | 前端访问地址 | `http://localhost:5173` | 生产 |
| `CORS_ALLOWED_ORIGINS` | CORS允许的源 | `http://localhost:5173` | 所有 |

### 可选环境变量
| 变量名 | 说明 | 默认值 | 环境 |
|--------|------|--------|------|
| `DB_SSL_ENABLED` | 是否启用SSL连接 | `false` | 生产 |
| `DB_POOL_MAX_SIZE` | 数据库连接池最大连接数 | `20` | 所有 |
| `LOG_LEVEL` | 日志级别 | `INFO` | 所有 |
| `FILE_UPLOAD_MAX_SIZE` | 文件上传最大大小(MB) | `10` | 所有 |
| `REDIS_HOST` | Redis缓存主机（如启用） | ` ` | 可选 |
| `REDIS_PORT` | Redis缓存端口 | `6379` | 可选 |
| `API_RATE_LIMIT` | API限流（请求/分钟） | `100` | 生产 |

### 配置文件说明
- **后端配置**：`application.yml`（基础配置）+ `application-{env}.yml`（环境特定配置）
- **前端配置**：`.env`（开发环境）+ `.env.production`（生产环境）
- **敏感配置**：通过环境变量注入，不写入配置文件

## 容器化部署

### Docker镜像构建
```dockerfile
# 前端Dockerfile示例
FROM nginx:alpine
COPY frontend/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/nginx.conf

# 后端Dockerfile示例
FROM openjdk:17-jdk-slim
COPY backend/target/cec-new-energy-code-1.0.0.jar /app.jar
ENTRYPOINT ["java", "-jar", "/app.jar"]
```

### Docker Compose部署（开发/测试）
```yaml
version: '3.8'
services:
  postgres:
    image: postgres:14
    environment:
      POSTGRES_DB: cec_new_energy_code
      POSTGRES_PASSWORD: your_password
    volumes:
      - postgres_data:/var/lib/postgresql/data
      - ./code/sql:/docker-entrypoint-initdb.d
    ports:
      - "5432:5432"

  backend:
    build: ./backend
    environment:
      DB_HOST: postgres
      DB_PORT: 5432
      DB_NAME: cec_new_energy_code
      DB_USER: postgres
      DB_PASSWORD: your_password
    ports:
      - "8080:8080"
    depends_on:
      - postgres

  frontend:
    build: ./frontend
    ports:
      - "80:80"
    depends_on:
      - backend
```

### Kubernetes部署（生产）
- **命名空间**：`cec-new-energy-code`
- **部署资源**：Deployment + Service + Ingress
- **配置管理**：ConfigMap + Secret
- **存储**：PersistentVolumeClaim（数据库）
- **网络**：Ingress + TLS证书

## 传统服务器部署

### 服务器要求
- **CPU**：2核以上
- **内存**：4GB以上
- **磁盘**：50GB以上（数据库单独考虑）
- **操作系统**：CentOS 7+/Ubuntu 20.04+

### 部署步骤
1. **环境准备**：安装JDK 17、Node.js、PostgreSQL、Nginx
2. **数据库部署**：安装配置PostgreSQL，创建数据库和用户
3. **后端部署**：
   ```bash
   # 上传JAR包
   scp target/cec-new-energy-code-1.0.0.jar user@server:/opt/app/
   
   # 创建systemd服务
   sudo vi /etc/systemd/system/cec-backend.service
   
   # 启动服务
   sudo systemctl start cec-backend
   sudo systemctl enable cec-backend
   ```

4. **前端部署**：
   ```bash
   # 构建前端
   pnpm build
   
   # 上传到Nginx目录
   scp -r dist/* user@server:/var/www/html/
   
   # 配置Nginx反向代理
   sudo vi /etc/nginx/conf.d/cec-frontend.conf
   
   # 重启Nginx
   sudo systemctl restart nginx
   ```

## 监控与运维

### 日志管理
- **日志级别**：DEBUG（开发）、INFO（测试）、WARN（生产）
- **日志格式**：JSON格式，便于ELK收集分析
- **日志输出**：控制台 + 文件滚动 + 远程收集（生产）
- **敏感信息**：脱敏处理，不记录密码、密钥等

### 健康检查
- **后端健康检查**：`GET /api/health`（返回服务状态、数据库连接状态）
- **前端健康检查**：静态资源可访问性检查
- **数据库健康检查**：连接池状态、查询性能监控

### 性能监控
- **应用指标**：JVM内存、GC情况、线程池状态
- **接口监控**：响应时间、QPS、错误率
- **数据库监控**：连接数、慢查询、锁等待
- **资源监控**：CPU、内存、磁盘、网络

### 告警规则
- **紧急告警**：服务不可用、数据库连接失败
- **重要告警**：接口错误率 > 5%、响应时间 > 5秒
- **一般告警**：磁盘使用率 > 80%、内存使用率 > 85%

## 备份与恢复

### 数据备份策略
- **数据库备份**：每日全量备份 + 每小时增量备份
- **备份保留**：最近7天每日备份 + 最近4周每周备份
- **备份验证**：定期恢复测试，确保备份可用性

### 恢复流程
1. **评估影响**：确定数据丢失范围和时间点
2. **停止服务**：暂停应用写入操作
3. **恢复数据**：从备份恢复数据库
4. **验证数据**：检查数据完整性和一致性
5. **恢复服务**：逐步恢复应用服务

### 灾难恢复
- **RTO（恢复时间目标）**：< 4小时
- **RPO（恢复点目标）**：< 1小时数据丢失
- **备份存储**：本地 + 异地双备份
