## 选型理由

### 后端技术选型

- **Node.js + Express**：相较于 Java 传统方案，Node.js 在快速原型开发、轻量级服务场景下效率更高。Express 作为最成熟的 Node.js Web 框架，生态丰富、社区活跃、学习曲线平缓，适合本项目作为培训案例。
- **TypeScript**：提供完整的类型系统，提升代码可维护性和开发体验。后端和前端统一使用 TypeScript，降低技术栈切换成本，共享类型定义。
- **pg（原生驱动）**：直接使用 PostgreSQL 原生驱动，通过参数化查询（`$1`, `$2`）保证 SQL 注入防护。相比 ORM 方案，学员能更直观地学习 SQL 编写与数据库交互。
- **crypto（内置模块）**：JWT Token 的签名和验证使用 Node.js 内置 `crypto` 模块实现 HS256 算法，不依赖额外 JWT 库，减少依赖体积。
- **archiver**：时序规则导出场景需要将多个 Excel 文件打包为 ZIP，archiver 是 Node.js 生态中最成熟的流式 ZIP 打包库。

### 前端技术选型

- **Vue 3 Composition API**：Vue 3 的组合式 API 提供了更灵活的逻辑复用方式，配合 TypeScript 可获得最佳类型支持。
- **Element Plus**：Vue 3 企业级组件库，组件丰富、文档完善、社区活跃，适合快速搭建后台管理类页面。
- **ECharts 6**：业界领先的数据可视化库，图表类型丰富、交互能力强，满足统计分析的各类图表需求。
- **Vite**：新一代前端构建工具，开发环境热更新速度快，构建产物体积小。
- **Pinia**：Vue 3 官方推荐的状态管理方案，TypeScript 支持良好，API 简洁。
- **Vue Router 4**：支持路由守卫，用于实现登录态检查与页面跳转控制。

### 工程体系

- **pnpm monorepo**：使用 pnpm workspace 管理多包项目，后端、前端、共享包放在同一仓库统一管理。相比 npm/yarn 节省磁盘空间、安装速度快、天然支持 monorepo。

## 版本要求
