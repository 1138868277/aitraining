# 接口设计

## 请求与响应约定

### 通用响应结构

```json
{
  "code": 0,
  "message": "success",
  "data": {},
  "timestamp": 1712300000000
}
```

- `code`：成功为数字 `0`；失败为字符串类型的错误码（如 `"DICT_LOAD_FAILED"`）
- `message`：响应消息，失败时包含用户可读的错误描述
- `data`：响应数据
- `timestamp`：时间戳（毫秒）

### 错误响应结构

```json
{
  "code": "DICT_LOAD_FAILED",
  "message": "筛选条件加载失败，请刷新重试",
  "data": null,
  "timestamp": 1712300000000
}
```

### 分页响应结构

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "items": [],
    "total": 100,
    "pageNum": 1,
    "pageSize": 20
  }
}
```

### 分页请求参数

| 参数 | 类型 | 说明 |
|------|------|------|
| pageNum | number | 页码（从 1 开始） |
| pageSize | number | 每页条数（默认 20，上限 100） |

### 文件上传

- 使用 `multipart/form-data` 格式
- 文件字段名：`file`
- 支持文件类型：`.xlsx`, `.xls`
- 最大文件大小：一般文件 10MB，全量测点导入和 TSR 导入 200MB

### 通用请求头

| 请求头 | 说明 |
|--------|------|
| `Content-Type` | `application/json` 或 `multipart/form-data` |
| `Authorization` | `Bearer <token>`，登录后获取的 JWT Token |
| `x-session-id` | 会话标识（UUID），用于草稿区数据隔离 |

### 认证说明

除登录接口（`/api/auth/login`）、健康检查（`/api/health`）、版本信息（`/api/version`）外，所有业务接口均需在请求头中携带 JWT Token。Token 通过登录接口获取，有效期 24 小时。Token 过期后服务端返回 401 状态码，前端自动跳转至登录页。

## 错误码分段

| 范围 | 模块 |
|------|------|
| 通用错误 | `VALIDATION_ERROR`, `SYSTEM_ERROR`, `AUTH_REQUIRED`, `FORBIDDEN` 等 |
| 编码生成 | `DICT_LOAD_FAILED`, `CONDITION_INCOMPLETE` 等 |
| 编码校验 | `VALIDATE_LIST_EMPTY`, `FILE_FORMAT_ERROR`, `VALIDATE_LIMIT_EXCEEDED` 等 |
| 统计分析 | `IMPORT_FILE_FORMAT_ERROR`, `STATISTICS_EMPTY` 等 |

完整错误码定义见 `code/packages/contracts/src/error-codes.ts`。

## 外部服务依赖

当前版本无外部服务依赖，所有功能均为本地实现。用户认证使用内置的 JWT 实现（基于 Node.js crypto 模块），不依赖外部 OAuth 服务。
