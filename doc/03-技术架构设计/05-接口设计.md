## 请求与响应约定

### 通用请求头
- `Content-Type: application/json`（JSON请求）
- `X-Request-ID: {uuid}`（请求追踪）
- `X-Tenant-ID: {tenantId}`（租户ID，单租户演示项目可忽略）
- `X-API-Key: {apiKey}`（可选，用于简单访问控制或未来扩展）

### 通用响应结构
```json
{
  "code": 0,
  "message": "success",
  "data": {},
  "timestamp": 1640995200000
}
```
- `code`: 响应码，0表示成功，非0表示错误
- `message`: 响应消息
- `data`: 响应数据
- `timestamp`: 时间戳（毫秒）

### 分页请求参数
```json
{
  "pageNum": 1,
  "pageSize": 20,
  "sortField": "createTime",
  "sortOrder": "DESC"
}
```

### 分页响应结构
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "list": [],
    "total": 100,
    "pageNum": 1,
    "pageSize": 20,
    "pages": 5
  }
}
```

### 错误响应
```json
{
  "code": 10001,
  "message": "参数校验失败",
  "data": {
    "field": "typeDomainCode",
    "reason": "不能为空"
  },
  "timestamp": 1640995200000
}
```

### 文件上传
- 使用 `multipart/form-data` 格式
- 文件字段名：`file`
- 支持文件类型：`.xlsx`, `.xls`
- 最大文件大小：10MB

## 参数校验规范

### 校验层级
1. **Controller层**：使用Bean Validation（@Valid）进行基础格式校验
2. **Service层**：进行业务逻辑校验和数据一致性校验
3. **Domain层**：封装核心业务规则校验

### 常用校验注解
- `@NotNull` / `@NotBlank`：非空校验
- `@Size`：长度校验
- `@Pattern`：正则表达式校验
- `@Min` / `@Max`：数值范围校验
- `@Email`：邮箱格式校验
- `@Digits`：数字精度校验

### 错误码分段
- `10xxx`：编码生成模块错误码
- `20xxx`：编码校验模块错误码  
- `30xxx`：统计分析模块错误码
- `40xxx`：字典管理模块错误码
- `50xxx`：系统管理模块错误码
- `90xxx`：通用错误码（参数校验、权限等）

## 外部服务依赖

### 1. 统一认证服务（未来扩展）
- **服务名称**：公司统一认证中心
- **接口**：用户登录、Token验证、权限校验
- **调用时机**：未来如需用户体系时的鉴权拦截器
- **依赖级别**：无依赖（当前版本不需要，预留扩展能力）
- **说明**：当前应用为内部工具，不需要用户登录认证。如需未来扩展多用户体系，可接入统一认证服务。

### 2. 文件存储服务
- **服务名称**：公司基础文件服务
- **接口**：文件上传、下载、临时链接生成
- **调用时机**：Excel文件上传校验、结果导出
- **依赖级别**：弱依赖（有降级方案，可本地存储）

### 3. 消息通知服务
- **服务名称**：公司消息中心
- **接口**：发送站内信、邮件通知
- **调用时机**：批量任务完成通知、异常告警
- **依赖级别**：弱依赖（无通知不影响核心功能）

### 4. 审计日志服务
- **服务名称**：操作审计平台
- **接口**：记录关键操作日志
- **调用时机**：编码生成、校验、导出等关键操作
- **依赖级别**：弱依赖（本地有日志记录，可异步同步）
