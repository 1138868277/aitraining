# 业务流程

## 主流程

1.编码生成流程：用户完成权限校验→系统加载 11 项字典筛选条件→用户逐次选择条件→实时生成编码→临时保存批量汇总→确认后导出 Excel / 保存入库并生成历史记录
2.编码校验流程：用户选择输入方式（手动 / 粘贴 / Excel）→提交编码列表→系统按字典规则自动校验→标记合规 / 异常并给出纠错建议→查看结果并导出校验报告
3.分析统计流程：系统拉取入库编码与校验异常数据→完成全局 / 维度统计计算→生成柱状图、饼图可视化展示→用户查看编码质量与分布数据

## 关键分支流程

- 分支1：编码生成多批次汇总分支：单批次生成并临时保存→重复多轮 “选条件→生成→暂存”→临时列表批量编辑→最终统一导出 / 入库
- 分支2：编码校验部分修正分支：完成全量校验→筛选异常编码→手动修改 / 采纳系统建议→对修正编码重新校验→仅导出合规编码

## 异常流程

异常场景 1：11 项字典数据加载失败（数据库 / 接口异常）
处理方式：前端弹出明确提示，禁止生成操作，提供刷新重试按钮，日志记录异常信息

异常场景 2：Excel 导入格式错误 / 数据超千行上限
处理方式：终止导入，提示 “文件格式不合法或数据量超出限制”，引导使用标准模板、控制数据行数后重新上传

异常场景 3：编码生成无结果（条件冲突 / 未选全）
处理方式：提示 “未生成有效编码，请检查筛选条件是否完整合规”，清空无效展示项

异常场景 4：导出 / 复制功能失败（权限 / 浏览器限制）
处理方式：提示 “操作失败，请检查权限或重试”，保留当前编码列表，支持重新发起操作

异常场景 5：权限校验失败（未登录 / 无访问权限）
处理方式：自动跳转登录页，提示 “无系统使用权限，请联系管理员授权”
