# 范围与边界

## 本期功能范围

### 功能模块

#### 1. 编码生成（code-generation）
- **编码生成**：根据 11 类字典筛选条件实时生成合规编码，支持单条生成和批量生成（多维度笛卡尔积组合）
- **草稿管理（临时区）**：生成的编码可保存至临时草稿区，支持查看草稿列表、单条删除、批量清空
- **最近筛选条件**：自动记录最近使用的筛选条件，支持快速复用
- **历史记录管理**：生成的编码可入库保存，支持历史记录的分页查看、单条删除、批量删除
- **Excel 导出**：支持编码生成结果和历史记录的 Excel 导出

#### 2. 编码校验与纠错（code-validation）
- **编码解析**：解析 31 位编码各段位对应的字典含义
- **单条校验**：输入单条编码进行合规校验
- **批量校验**：粘贴多条编码进行批量校验
- **Excel 导入校验**：上传 Excel 文件批量导入编码进行校验
- **任务管理**：异步校验任务的状态跟踪与结果查询
- **纠错建议**：对异常编码基于编辑距离算法计算最接近的正确编码，给出修正建议
- **批量修正**：一键采纳建议进行批量修正
- **修正确认**：对修正后的编码逐条确认
- **校验结果导出**：将校验结果（含原因和建议）导出为 Excel
- **模板下载**：下载批量导入的标准 Excel 模板
- **编码映射保存**：将纠错后的编码映射关系入库保存

#### 3. 字典管理（dict）
- **字典树**：分层树形结构展示全部 11 类字典项
- **快速检索**：关键字模糊搜索字典项
- **编码解析**：将 31 位测点编码反向解析为各段位的字典含义
- **字典项浏览**：按字典类型分页查看字典项列表
- **手动新增字典**：支持手动新增和批量新增字典项，标记为手工添加
- **数据类码筛选**：按数据类型获取对应的数据类码
- **二级类码筛选**：按设备类型（风电/光伏/水电）获取对应的二级类码
- **字典统计**：字典概览统计、新增字典记录、类型域分布统计

#### 4. 场站管理（station）
- **场站列表**：分页查看场站信息
- **新增场站**：单个新增场站
- **批量新增**：批量导入新增场站
- **编辑场站**：修改场站信息
- **删除场站**：单个删除和全部清空（含确认提示）
- **Excel 导出**：场站列表导出

#### 5. 统计分析（statistics）
- **编码生成统计**：
  - 概览卡片：累计生成总数、今日/本周/本月生成数
  - 类型维度：风电、光伏、水电、不区分类型的编码数量统计
  - 二级类码维度：按设备类型筛选，横向柱状图展示
  - 场站维度：按场站聚合的横向柱状图
  - 分组详情列表：按生成批次/分组展示，支持展开查看单条编码、多选导出、批量删除
  - 生成趋势：编码生成量的时间趋势
- **全量测点统计**：
  - Excel 导入：上传存量测点编码文件，支持进度条与预估剩余时间，可取消导入
  - 概览卡片：总测点数、风电/光伏/其他分类统计
  - 二级类码维度：按设备类型筛选的横向柱状图
  - 场站维度：按场站聚合的横向柱状图
  - 数据清空：一键清空所有导入的测点数据
- **字典统计**：
  - 字典概览：各类型字典项数量统计
  - 新增字典：手动新增字典记录统计
  - 类型域分布：字典在风电/光伏域下的分布统计

#### 6. 数据源管理（datasource）
- **配置查看**：查看当前 PostgreSQL 数据源连接配置
- **连接测试**：测试配置的连接是否可用
- **保存切换**：保存新配置并在运行时动态切换数据源，无需重启服务
- **锁定保护**：配置表单默认锁定，防止误修改

## 非本期范围

- 用户认证与权限管理：当前未接入 Auth 权限系统，无登录页，无鉴权逻辑
- 操作审计日志：编码操作未记录操作人信息
- 编码规则可视化配置：编码规则为固定算法，未提供前端规则配置界面
- 字典树编辑：字典树为只读展示，字典项通过手动新增接口添加
- 移动端适配：当前仅支持桌面端浏览器

## 边界说明

- 技术栈：Node.js/Express（TypeScript）后端 + Vue 3（Element Plus）前端 + PostgreSQL 数据库
- 临时数据：草稿数据存储于后端内存，服务重启后丢失；校验任务数据存储于后端内存
- 持久化数据：历史编码记录、字典数据、场站数据、统计导入数据持久化于 PostgreSQL
- 数据源配置：持久化于 `config/datasource.json` 文件，修改后运行时生效
- 数据库设计：表间无外键约束，数据完整性由应用层保证；采用逻辑删除（`if_delete` 字段）
