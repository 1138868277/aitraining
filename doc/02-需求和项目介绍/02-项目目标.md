# 项目目标

## 总体目标

- 业务层面：构建一套覆盖"生成 - 解析 - 修正 - 稽核 - 字典 - 统计 - 规则"全流程的测点编码管理工具，统一华电新能源场站编码标准，解决编码规则复杂、人工易出错、跨部门沟通成本高的核心痛点。
- 技术层面：完成基于 Node.js/Express（TypeScript）+ Vue 3（Element Plus）技术栈的工具开发，覆盖编码生成引擎、编码解析、编码修正、重复编码稽核、自动编码匹配、字典管理、场站管理、统计分析、字典审批、动态数据源切换、用户认证、时序稽核规则生成等核心能力。
- 标准化层面：沉淀测点编码规则算法与可视化配置逻辑，形成可复用的企业级编码规范模板，降低后续新场站、新设备的编码实施周期。

## 本期实现目标

- **用户认证**：支持用户名+密码登录，JWT Token 鉴权，未登录用户自动跳转登录页；支持多租户（集团/云南省/福建省）数据隔离。
- **用户管理**：集团管理员可新增、编辑、删除用户，每个用户归属于指定区域。
- **编码生成**：实现基于 11 类字典条件的编码实时生成引擎，支持单条/批量生成、临时草稿管理、最近筛选条件记录、历史记录管理、Excel 导出与数据库入库。内置快速检索面板支持数据码模糊搜索。
- **自动编码匹配**：支持上传 Excel 文件，通过列映射向导将文件字段映射到系统字典字段，自动匹配生成合规编码。
- **编码解析**：实现 31 位编码反向解析，分段展示各段位对应的字典含义及识别状态，支持单条和批量两种解析模式。
- **编码修正**：实现 Excel 导入修正数据，批量对编码的指定段位进行修改，修正前后逐段对比展示，自动检测修正后编码是否与全量测点重复，支持结果导出。
- **重复编码稽核**：实现批量粘贴编码列表，自动检测每条编码是否已在全量测点中存在，标记重复状态。
- **字典管理**：实现 11 类字典数据的分层树形展示、懒加载、快速检索，以及手动新增字典项、新增记录查询与导出。支持数据码查询和树形浏览两种模式。
- **字典审批**：区域用户可提交数据码/数据类码新增申请，集团管理员可审批通过或驳回，支持按区域和状态筛选查询。
- **场站管理**：实现场站数据的增删改查、批量新增、全部清空和 Excel 导出。
- **分析统计**：
  - 编码生成统计：概览卡片（累计/今日/本周/本月）、类型/二级类码/场站维度图表、分组详情列表。
  - 全量测点统计：Excel 大文件导入（含进度条与取消）、概览/二级类码/场站维度图表、测点列表查询。
- **时序稽核规则**：实现场站数据与测点数据导入，自动生成死值（sz）、突变（tb）、越限（yx）、终端（zd）四类时序稽核规则，支持单类型 Excel 导出、全部类型 ZIP 打包导出、分批拆分导出。流水线式操作步骤依赖控制（每步需上一步完成后才可操作），页面刷新自动恢复导入/生成进度，上传进度在刷新后正确保持 100%。
- **数据源管理**：实现 PostgreSQL 数据源配置的查看、连接测试、保存与运行时动态切换，配置表单含锁定保护机制。
