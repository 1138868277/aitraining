# 项目目标

## 总体目标

- 业务层面：构建一套覆盖"生成 - 校验 - 修正 - 稽核 - 字典 - 统计"全流程的测点编码管理工具，统一云南区域新能源场站编码标准，解决编码规则复杂、人工易出错、跨部门沟通成本高的核心痛点。
- 技术层面：完成基于 Node.js/Express（TypeScript）+ Vue 3（Element Plus）技术栈的工具开发，覆盖编码生成引擎、批量校验、编码修正、重复编码稽核、字典管理、场站管理、统计分析、动态数据源切换等核心能力。
- 标准化层面：沉淀测点编码规则算法与可视化配置逻辑，形成可复用的企业级编码规范模板，降低后续新场站、新设备的编码实施周期。

## 本期实现目标

- 编码生成：实现基于 11 类字典条件的编码实时生成引擎，支持单条/批量生成、临时草稿管理、最近筛选条件记录、历史记录管理、Excel 导出与数据库入库。
- 编码校验与纠错：实现单条/批量/Excel 导入三种校验方式，系统按字典规则自动校验，标记合规/异常并给出纠错建议，支持批量修正与纠错确认。
- 编码解析：实现 31 位编码反向解析，分段展示各段位对应的字典含义及识别状态。
- 编码修正：实现 Excel 导入修正数据，批量对编码的指定段位进行修改，修正前后逐段对比展示，自动检测修正后编码是否与全量测点重复，支持结果导出。
- 重复编码稽核：实现批量粘贴编码列表，自动检测每条编码是否已在全量测点中存在，标记重复状态。
- 字典管理：实现 11 类字典数据的分层树形展示、懒加载、快速检索，以及手动新增字典项、新增记录查询与导出。
- 场站管理：实现场站数据的增删改查、批量新增、全部清空和 Excel 导出。
- 分析统计：
  - 编码生成统计：概览卡片（累计/今日/本周/本月）、类型/二级类码/场站维度图表、分组详情列表。
  - 全量测点统计：Excel 大文件导入（含进度条与取消）、概览/二级类码/场站维度图表、测点列表查询。
  - 字典统计：字典概览统计、新增字典记录统计、类型域分布统计。
- 数据源管理：实现 PostgreSQL 数据源配置的查看、连接测试、保存与运行时动态切换，配置表单含锁定保护机制。
