# 用户与场景

## 目标用户

- 用户类型：
    1. 华电新能源测点编码专职工作人员
    2. 新能源场站设备管理辅助人员
    3. 公司数据标准与运维管控人员
- 用户特征：
    1. 熟悉新能源发电设备业务，掌握基础编码规则，无专业技术开发背景
    2. 日常高频处理单条/批量编码编制、核对、修改、上报与整改
    3. 对编码准确性、合规性、可追溯性要求严格
    4. 习惯使用公司内部统一系统，偏好简洁、少步骤、易上手的操作
    5. 需定期输出编码统计报表，用于工作汇报与管理复盘

## 核心使用场景

- 场景1：新建设备批量编码生成
    编码人员按字典条件筛选，实时生成合规编码（单条或批量组合），可多次临时保存草稿后汇总，或直接入库保存。支持基于最近筛选条件快速复用地生成。最终导出 Excel 或入库留存，用于新建场站/设备的测点编码编制。

- 场景2：历史存量编码合规校验和修改
    工作人员输入单条编码、粘贴编码列表或上传 Excel 文件，系统自动校验并标记对错，对错误编码给出原因与修正建议（基于编辑距离的智能纠错），支持批量修正与逐条确认，导出结果用于存量编码整改。

- 场景3：编码解析
    工作人员输入 31 位编码，系统自动解析各段位对应的字典含义，以分块卡片和编码结构图展示，辅助编码学习、排错与培训。

- 场景4：批量编码修正
    工作人员上传含"测点编码、测点描述、修改内容"的 Excel 文件，系统按修改内容解析出需要修改的段位（如二级类码、数据类码、数据码等），批量生成新的修正编码，自动检测修正后编码是否已在全量测点中重复，展示修改前后分段对比，最终导出修正结果。

- 场景5：重复编码稽核
    工作人员粘贴一批待新增的编码列表，系统自动与数据库全量测点数据进行比对，逐条标记"重复"或"不重复"，确保新增编码的唯一性。

- 场景6：字典查询与手动新增
    工作人员通过分层树形结构浏览全部字典项，使用快速检索定位特定字典条目，对缺失字典项可手动新增。支持查看手动新增字典记录并导出。

- 场景7：场站信息管理
    工作人员维护新能源场站基础数据，支持单条新增、批量导入、编辑修改和删除，支持全部清空和 Excel 导出备份。

- 场景8：编码生成统计与管理汇报
    管控人员查看编码生成概览（累计/今日/本周/本月生成量），按类型、二级类码、场站等维度查看图表分布，查看分组详情列表，用于编码质量监控、标准落地复盘与工作汇报。

- 场景9：全量测点数据导入与分析
    管控人员上传存量测点编码 Excel 文件，系统自动解析导入（含进度提示与取消能力），导入完成后按类型、二级类码、场站维度进行统计分析和图表可视化展示。

- 场景10：动态切换数据源
    运维人员在系统设置中配置新的 PostgreSQL 数据源连接信息，测试连接成功后保存并生效，系统运行时动态切换数据库，无需重启服务。

## 用户关键诉求
    1. 操作极简：界面清晰、步骤少，无需培训即可快速使用
    2. 生成零错：严格按公司字典规则生成，彻底避免人工编码错误
    3. 批量高效：支持千行级导入导出、批量校验与批量修正，大幅提升作业效率
    4. 智能纠错：异常编码自动提示并给出修正方案，减少排查耗时
    5. 数据可溯：生成记录可保存、可查询，满足管理与审计要求
    6. 统计直观：图表可视化呈现合规情况，快速支撑管理决策
    7. 字典透明：可随时查阅全部字典项，支持手动补充缺失字典
    8. 编码唯一：新增编码前可批量稽核是否与存量测点重复
    9. 配置灵活：数据源可动态切换，适应多环境部署需要
